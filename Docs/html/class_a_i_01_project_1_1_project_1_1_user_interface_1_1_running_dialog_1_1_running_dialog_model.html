Detailed Description

Process methods of an algorithm according to requests of the dialog.

The sequence of processing an algorithm:

  1. Load - load the data from csv files to self.data_matrix self.test_data_matrix
  2. Normalize - Show a dialog for designing the algorithm data and produce a normalized self.algorithm_data and self.test_algorithm_data according to the design
  3. Init - Activate the init method of the train
  4. Step/Run to end - Process the algorithm
  5. Test - test the algorithm with test data
  6. Scipy - Activate the scipy version of the algorithm
  7. ScikitLearn - Activate the scikit - Learn of the algorithm

Operating method

  1. The run method is activated by the dialog with parameter that tells which processing step is requested
  2. The run method activate a thread with thread_main as its starting point
  3. The thread_main method activates the requested method (for example load, init, step etc.)
  4. The requested method returns 4 results which are set as members of the module
    1. self.success - If the processing succeeded
    2. self.result - The result of the method
    3. self.end_message - A return message from the method
    4. self.algorithm_message - A return message from the algorithm
  5. After getting the results the thread_main set an event to signal to the dialog signaling that it finished

Summary of connections between the dialog and the model

Variables created by the dialog

Variables used for all the actions

  1. self.view.event - The event is initialized by the dialog and given to model in the run method. The event is set by the model to signal to the dialog that the processing ended
  2. self.view.messages - A queue with messages from the model to be presented on the dialog

Variables used for the load action

  1. self.train - The training class name (which is converted by the method to an object from the class)
  2. self.algorithm - The algorithm class name (which is converted by the method to an object from the class)
  3. self.filename - The file name to load the algorithm from
  4. self.test_filename - The file name to load the test data

Variables used by the normalized actions

  1. self.normalize_data - The normalize data as designed by the AlgorithmDataDesign dialog
  2. self.data_matrix - The data matrix as generated by the AlgorithmDataDesign dialog
  3. self.test_data_matrix - The test data matrix as generated in the dialog using the same AlgorithmDataDesign dialog
#### Variables used in the step/run_to_end actions
-#  self.num_steps - The maximum number of steps to process in one step method
                     or the number of steps between 2 updates of the plot in run to end

#### Variables used in the test action
-   No new variables are needed for this action from the dialog

#### Variables used in the scipy action
-   No new variables are needed for this action from the dialog

#### Variables used in the scikitLearn action
-   No new variables are needed for this action from the dialog

### Variables created by the model
-#  self.success - Informs the dialog if the algorithm succeeded
-#  self.result - The result of the running
-#  self.end_message - A message produced by the model to inform ending of the processing
-#  self.algorithm_message - A message produced by the algorithm informing the result of the processing
-#  self.enable_actions - A dictionary that tells the dialog which actions are possible after processing
                          the current step

Activate the scipy version of the algorithm.

  • The scipy action is using the same data as the algorithm
  • The unchanged data is found in the self.data_matrix and self.test_data_matrix
