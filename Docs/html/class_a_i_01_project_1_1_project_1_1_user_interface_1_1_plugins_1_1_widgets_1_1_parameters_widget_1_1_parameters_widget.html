Detailed Description

This widget is responsible for presenting and changing attributes.

Dialog Description

Perpose

To set parameters for a class

Structure

  1. A main combo box for selecting the target class for the parameter
  2. A Grid in scroll area with 2 columns:
    1. The parameter name
    2. The parameter setting widget
  3. A horizontal layout with the buttons:
    1. Reset
    2. Load
    3. Save
    4. A LineEdit with the name of the file

Widget working method

  1. After the widget was loaded the method init is called
  2. The init gets 2 main parameters:
    1. The base class
    2. The label for the widget
  3. The init gets all the inheritors of the base class saves them and fill there names in the MainComboBox
  4. When the MainComboBox is filled the slot of the ComboBox is activated
  5. The slot of the combo box gets the parameters of the target class by importing it and activating the prms method of it
  6. The slot calls the StdPrmInput.widgets to creates the widgets for each parameters and filles the grid
  7. Now the user can change the values of the parameters
  8. The attributes can also be changed by the buttons
  9. If the user changes the selection in the MainComboBox the slot method start again

File save and load handling

The file structure

  1. The first line of the file is the names (keys) of the attributes
  2. The second line of the file is a standared attribute that contains the target class
  3. The rest of the lines are the values of the standared parameters

The loading

  1. Check if the module is the same between the MainComboBox and the file (second line)
  2. Change the values of the parameters. The changing is done in the following way:
    1. The first column is the name of the parameter
    2. The second column is the value of the parameter
    3. The insertion is : model[Name]["value"] (using indexers on the ParametersList and Parameter)

Attributes

  1. filename - The name of the file of the last save or load operation
  2. noSaveQuestion - Used to set is a question for saving should be done
  3. model - The ParametersList

Activation Parameters
