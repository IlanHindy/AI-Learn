Detailed Description

running dialog for all the algorithms

Dialog Description

Purpose

To run an AI algorithms

Structure

The dialog is composed from

  1. Horizontal layout that holds 2 Plot containers (viewing of a graph) and a text edit for the messages
  2. Grid layout with 4 rows (from top to button):
    1. labels that will be used to show the time elapse for the action activated by the buttons below them
    2. The buttons row to activate the algorithm
    3. labels that will be used to show the time elapse for independent actions
    4. The buttons row for the independent actions

Dialog working method

  1. Each one of the buttons is responsible for one step in the processing of the algorithm
  2. The steps are :
    1. Load
    2. Normalize
    3. Init
    4. Repeating steps or Run to end
    5. Test
  3. Additionally There are independent actions:
    1. Activate the scipy version of the algorithm
    2. Activating the scikit - Learn version of the algorithm
  4. Each button is doing:
    1. Get all the inputs needed for the action
    2. Activate the activate method
  5. The activate method does:
    1. Start a timer
    2. Call the run method of the model to perform the action
  6. While the action is running in a separate thread in the model, the timer is active in the dialog
  7. The timer is running as long as the model is running an action
  8. When the model finished an action, it sets an event
  9. The timer method checks the event and if it is set it does:
    1. Stop the timer
    2. Generate a finish/error message
    3. Set the enable attribute of the buttons according a dictionary in the model

Attributes

  1. self.model - The model of the dialog (That performs the actual operations)
  2. self.timer - The timer that is presenting the time and activates the end action operations
  3. self.action - The action to perform (set by the activate method to be sent to the model)
  4. self.timerLabel - Tells the timer event handler where to write the time elapsed when an action is performed
  5. self.timerText - A text for the time label (set by the action method and used by the timer event handler)
  6. self.startTime - The start time of the action to be used by the timer event
  7. self.stopEvent - The event that will be used by the model to signal to the timer that it finished
Parameters
Parent(QDialog) - The calling dialog

Generate the end action message.

The inputs of the message generation is found in the model

The message is generated in the following way:

  1. If the algorithm message is not empty add the algorithm message to the model message
  2. Else leave the model message
  3. If the step succeeded produce an information message
  4. Else produce an error message
